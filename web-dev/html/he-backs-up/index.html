파일과 데이터베이스를 통째로 로컬 서버 또는 Google Drive에 백업하고, 언제든 원하는 시점으로 되돌릴 수 있어요.

v1.0.0 · WordPress 5.8+ · PHP 7.4+
주요 기능

백업에 필요한 모든 것

복잡한 설정 없이 클릭 한 번으로 사이트 전체를 백업하세요.

📦

전체 백업

wp-content 디렉토리 전체와 MySQL 데이터베이스를 하나의 ZIP 파일로 압축합니다. 테마, 플러그인, 미디어 파일 모두 포함.

☁️

Google Drive 저장

버튼 하나로 Google 계정과 연결. 백업 파일이 자동으로 내 Google Drive에 업로드됩니다. 서버 용량 걱정 없음.

🖥️

로컬 서버 저장

서버의 지정된 디렉토리에 백업 파일을 보관합니다. 보존 개수를 설정하면 오래된 파일이 자동으로 정리됩니다.

⏰

자동 예약 백업

매일 · 매주 · 매월 원하는 주기를 설정하면 자동으로 백업이 실행됩니다. WordPress Cron 기반으로 설치 후 바로 작동.

🔄

원클릭 복구

저장된 백업 목록에서 원하는 시점을 선택해 복구. 로컬 또는 Google Drive의 백업 파일 중 선택할 수 있습니다.

🔒

보안 설계

Google 토큰은 AES-256 암호화 후 저장. 모든 요청에 WordPress Nonce 검증. 백업 파일은 웹에서 직접 접근 불가.

작동 방식

3단계로 끝나는 백업

1

플러그인 설치

ZIP 파일을 WordPress 관리자에서 업로드하거나 plugins 폴더에 압축 해제

2

저장소 설정

로컬 서버, Google Drive, 또는 둘 다 활성화. Google Drive는 버튼 클릭 한 번으로 연결

3

백업 실행

지금 바로 백업하거나 자동 예약 설정. 완료 후 백업 목록에서 복구도 가능

Google Drive 연동

서버 용량이
걱정된다면

10MB씩 나눠서 업로드하는 청크 방식으로 서버에 임시 파일이 최소화됩니다.

  • ✓
    버튼 하나로 Google 계정 연결 — 직접 API 키 발급 불필요
  • ✓
    내 Google Drive에만 저장 — 개발자도 접근 불가
  • ✓
    drive.file 스코프만 사용 — 기존 파일 절대 건드리지 않음
  • ✓
    토큰 자동 갱신 — 한 번 연결하면 재인증 불필요
  • ✓
    Google Drive 전용 모드 시 서버에 영구 사본 없음

설치 가이드

5분이면 시작할 수 있어요

📦 방법 1 — 관리자에서 직접 설치

  1. GitHub에서 he-backs-up.zip 다운로드
  2. WordPress 관리자 → 플러그인 → 새 플러그인 추가
  3. "플러그인 업로드" 클릭 후 ZIP 파일 선택
  4. 설치 후 플러그인 활성화
  5. 왼쪽 메뉴 "He Backs Up" 에서 설정 시작

🖥️ 방법 2 — FTP / 파일 매니저

  1. ZIP 파일을 로컬에서 압축 해제
  2. he-backs-up 폴더를 서버에 업로드
  3. 업로드 경로:
    4. wp-content/plugins/he-backs-up/
  4. WordPress 관리자 → 플러그인에서 활성화
